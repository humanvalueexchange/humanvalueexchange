# Executive Notice: HVE GitHub Org Repo Is Now Live

**Date:** June 4, 2026  
**Version:** 1.0  
**From:** Hans Westphal, CEO  
**To:** Atlas, Mika, Claude, Hermes, Mercury, Apollo, Copilot EA, HVE-jrdev, and all active collaborators  
**Labels:** executive, org-update, github, migration

---

Team,

The Human Value Exchange GitHub organization repository is now live:

**https://github.com/humanvalueexchange/humanvalueexchange**

This is the start of our transition from an individual-account-centered GitHub footprint to an organization-centered one.

## What Changes Now

- **`github.com/humanvalueexchange` is the default GitHub home for HVE going forward**
- The new org repo is now the destination for curated, high-value HVE operating documents
- We are migrating selectively - **gold only, not everything**
- We will continue using pull requests and visible review as the default workflow

## First Migration Batch Completed

The first approved canon set has already been moved into the org repo on a migration pull request:

- HVE org chart v4.0
- GitHub control-plane memo
- Company briefing document v2.0
- Programs brief

This gives us a clean starting core inside the org namespace without dragging over lower-value or outdated material.

## Operating Guidance

Effective immediately:

1. Prefer links and references under **`https://github.com/humanvalueexchange`** for new HVE coordination.
2. Treat the org repo as the strategic long-term home for core HVE records.
3. Continue to migrate only the best, highest-signal documents first.
4. Preserve reviewability and audit trail through branches and pull requests.

## Why This Matters

This move improves:

- **Identity** - HVE work now lives under HVE's own GitHub organization
- **Durability** - core records are being consolidated in the right long-term home
- **Clarity** - the team now has a cleaner canonical namespace to reference
- **Sovereignty** - this aligns our repo structure with the broader push toward owned infrastructure and tighter operational control

## Transition Note

We are still in transition, so some bumpiness is expected for a short period.

That is acceptable.

The rule is simple: when in doubt, orient new durable HVE work toward the org namespace and migrate legacy material selectively, not blindly.

## Closing

This is an infrastructure and operating-system upgrade for the company, not just a repo rename.

We now have the right GitHub home to build from.

— Hans Westphal, CEO
